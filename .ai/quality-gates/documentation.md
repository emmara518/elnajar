# Documentation Gate

## Purpose
Ensure documentation remains accurate and synchronized with implementation.

## Checks
- [ ] All new features documented
- [ ] API changes reflected in API docs
- [ ] Schema changes reflected in database docs
- [ ] Architecture changes reflected in architecture docs
- [ ] No contradiction between code and docs
- [ ] Cross-references verified
- [ ] Release notes updated (if applicable)

## Gatekeeper
Documentation Agent

## Failure
Block merge. Docs must match implementation.

## Override
Only Lead Engineer may override with documented follow-up plan.

# Documentation Review Checklist

- [ ] All new features documented
- [ ] API changes reflected in docs
- [ ] Schema changes reflected in docs
- [ ] Architecture changes reflected in docs
- [ ] ADR written (if architecture changed)
- [ ] No contradiction between code and docs
- [ ] Cross-references verified (links valid)
- [ ] Release notes updated (if applicable)
- [ ] Deprecated patterns removed from docs
- [ ] Spelling and grammar checked
- [ ] Examples are correct and up-to-date
